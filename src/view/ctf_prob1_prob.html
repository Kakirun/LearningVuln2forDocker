小手調べSQLインジェクション

シナリオ：

 ある企業の古い社内管理システムを発見しました。 このシステムには、管理者（Admin）専用のログインページがあるようです。

あなたの目的：

 あなたのアカウント情報は不明です。 しかし、どうやらこのログインフォームには、先ほど学習したSQLインジェクションの脆弱性があるようです。 ユーザー名とパスワードを正しく入力せずに、学んだ知識を利用して認証を回避し、管理者としてログインしてください。

成功条件：

 管理者アカウントのパスワードが今回のフラグです。フラグは { } で囲まれた文字列で、{ } も含めて答えを入力してください。

ヒント1

SQLインジェクションの原理

学習ページの「実際にコマンドを入力してみる」で成功したコマンドを思い出してください。
パスワードを知らなくても、「'1'='1'（常に正しい）」 というルールを無理やり成立させれば、コンピュータを騙すことができます。

ヒント2

シングルクォートの利用

'1'='1'を単に入力するだけでなく、SQLクエリ全体でどのような処理になるか思い出しましょう。
ORとシングルクォート（'）を上手く利用することが大切です。
忘れた場合は学習パートを見てみましょう。
